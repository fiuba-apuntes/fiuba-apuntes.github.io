Informar errores o sugerir mejoras

Lo más fácil es informarnos los errores ortográficos, errores en las fórmulas, gráficos faltantes. También se pueden sugerir contenidos nuevos, aportar gráficos de mejor calidad, etc.

Para informarnos o sugerirnos cosas, contacta con algún administrador del proyecto. Otra opción es crearte una cuenta o iniciar sesión en GitHub y crear un issue en el repositorio correspondiente al apunte sobre el cual se va a hacer el reporte. Los enlaces a los repositorios los encontrarás en la página de inicio.

Proporcionar mejoras mediante un fork

GitHub proporciona una forma relativamente sencilla para realizar forks para poder realizar cambios al proyecto. Luego de hacer las correcciones y/o mejoras sólo hay que solicitar un pull request, es decir, una peticion para unir los cambios al repositorio principal. Un administrador del repositorio revisará los cambios realizados y los aprobará.

Colaborar directamente en proyecto

Si tu colaboracion mediante forks es frecuente, podemos decidir darte acceso directo al proyecto para que colabores y puedas, no sólo aportar tus sugerencias, implementar las de otros colaboradores que no tengan acceso a los repositorios del proyecto.
